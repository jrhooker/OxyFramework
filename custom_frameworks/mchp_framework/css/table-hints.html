Hints for table elements

nozebra

This table has at least one outputclass attribute value. The detected value of nozebra means that when the table is published, it will not have the alternating gray rows. There may be more values defined; this note only detects the first value.

nocolor

This table has at least one outputclass attribute value. The detected value of nocolor
means that when the table is published, the body rows of the table will have no background. color.
There may be more values defined; this note only detects the first value.
See more values available for the outputclass attribute here [?]

scale

This table has a manually defined scale attribute value. This will scale the text content of the table to
the percentage defined This means that when the table is published, the body rows of the table be the percentage of the default body text size.
The default treatment for table content is 80% of default body text size,
, so any value smaller than 80 shrink the table text. See more values available for the here [?]

compact

This table has at least one outputclass attribute value defined. The detected value of of compact
means that when the table is published, the body rows of the table will have less padding in order to make the table more compact.
There may be more values defined; this note only detects the first value.
See more values available for the outputclass attribute here [?]

frame

This table has a frame attribute value of none.
This means that when the table is published, the outer edges of the table
will have no border. See more values available for the table elements here here [?]

headercolumn

This table has at least one outputclass attribute value. The detected value of headercolumn
means that when the table is published, the first column of the table will be styled as a header.
There may be more values defined; this note only detects the first value.
See more values available for the outputclass attribute here [?]

widetable

This table has an outputclass attribute value of widetable. This means that when the table is published, the table will push into the left margin in order to
be as wide on the page as possible without rotating either the table or the page.
See more values available for the outputclass attribute here [?]

fitcontent

This table has an outputclass attribute value of fitcontent. This means that when the table is published, columns will take up the minimum
space needed for their content, resulting in most cases in a table which does not span the width of the page.
This is mostly used for tables of numbers or small values.
There may be more values defined; this note only detects the first value.
See more values available for the outputclass attribute here [?]

widetablenozebra

This table has an outputclass attribute value of widetablenozebra. This means that when the table is published, the table will push into the left margin in order
to be as wide on the page as possible without rotating either the table or the page and the alternating gray and white horizontal rows will not be present.
There may be more values defined; this note only detects the first value.
See more values available for the outputclass attribute here [?]

rotatepage

This table has an outputclass attribute value of rotatepage. This means that when the table is published, the page will be rotated to landscape orientation.
There may be more values defined; this note only detects the first value.
If the widetable value is in the outputclass attribute as well, then the table will be taking up the maximum space available on the page.
See more values available for the outputclass attribute here [?]

landscapetable

This table has an outputclass attribute value of landscapetable. This means that when the table is published, the table will be rotated on the page 90 degrees counter-clockwise.
There may be more values defined; this note only detects the first value.
